# Дизайн: общие фильтры тренировок и масштаб графика

## Цель

Сделать так, чтобы вкладки `Тренировки` и `График` использовали одну выбранную фильтрацию по периоду и типу тренировки. Добавить множественный выбор типов тренировок, последние пять пользовательских периодов и управление масштабом графика по оси X через кнопки `-`, reset и `+`.

## Состояние после коммита расстояний

Коммит `a5c511d fix: correct Garmin distance units` уже зафиксировал корректную конвертацию Garmin `distance` из сантиметров в метры/километры. Новая работа строится поверх него.

## Общая модель фильтров

Фильтры живут на уровне `SportDashboardView`, выше обеих вкладок:

- выбранный период;
- выбранные типы тренировок;
- последние пять пользовательских периодов;
- масштаб графика по X.

`SportDashboardView` один раз загружает bundled Garmin JSON и передает дочерним вкладкам уже общее состояние фильтров. Новые файлы с Garmin-данными не создаются.

## Период

Период по умолчанию: `1 неделя` от последней доступной активности в bundled Garmin JSON.

Доступные пресеты:

- `1 неделя`;
- `2 недели`;
- `1 месяц`;
- `3 месяца`;
- `6 месяцев`;
- `1 год`.

Доступен пользовательский период через date range. Когда пользователь нажимает `Готово` для пользовательского периода, этот период сохраняется в список последних. Хранится максимум пять последних пользовательских периодов. В UI они показываются отдельной секцией `Последние`.

Если пользователь повторно выбирает уже существующий пользовательский период, он переносится вверх списка последних.

## Типы тренировок

Фильтр типов тренировок поддерживает множественный выбор. По умолчанию выбраны все доступные типы из текущего Garmin export.

Пункты выбора показываются человекочитаемыми категориями:

- `Плавание Море`;
- `Бассейн`;
- `Велик`;
- `Бег`;
- `Ходьба`;
- `Силовая`;
- `Эллипс`;
- `SUP`;
- `Отдых`;
- название Garmin для прочих типов.

`cycling` и `indoor_cycling` объединяются в одну категорию `Велик`. Фильтр применяется одинаково к списку карточек и к расчету графика. При этом существующая защита графика от невалидных записей, meditation, track me и rest-like активностей сохраняется: общий фильтр типов выбирает кандидатов, а расчет усилия дополнительно использует `TrainingActivityFilter.default`.

## Вкладка `Тренировки`

Список карточек строится только из активностей внутри выбранного периода и выбранных типов. Сортировка остается от новых к старым.

Строка над списком показывает выбранный период и краткое состояние фильтра типов. Иконка периода открывает sheet периода. Отдельная иконка фильтра открывает sheet множественного выбора типов.

Настройки отображаемых полей карточек по типам активности остаются как сейчас.

## Вкладка `График`

Текст `ты тренируешься уже 50 недель` заменяется на выбранный период, такой же как во вкладке `Тренировки`.

График считает точки только по выбранному периоду и выбранным типам. Еженедельная гранулярность остается строго понедельник-воскресенье. Неполные недели в начале и конце выбранного периода отбрасываются существующей логикой расчета.

Формат дат по оси X: `dd/MM/yyyy`, например `07/06/2026`.

Количество подписей по X рассчитывается от видимой ширины и текущего масштаба: на ширину экрана iPhone должно попадать не больше четырех подписей. При горизонтальном скролле подписи повторяются с таким шагом, чтобы сохранялась эта плотность.

## Масштаб графика

Над графиком добавляется компактное управление:

- кнопка `-` уменьшает расстояние между недельными точками;
- центральная кнопка reset возвращает исходный масштаб;
- кнопка `+` увеличивает расстояние между недельными точками.

Исходный масштаб соответствует текущему визуальному масштабу графика. Reset disabled или визуально тихий, когда график уже в исходном масштабе. `-` и `+` disabled на минимальном и максимальном масштабе.

## Архитектура

Новые и расширенные доменные единицы:

- `TrainingPeriodSelection` получает человекочитаемый `title`, публичный `dateRange` и Codable-представление для recent custom periods;
- `TrainingActivityTypeCategory` нормализует Garmin activity type в человекочитаемую категорию;
- `TrainingActivityTypeSelection` хранит выбранные категории и проверяет включение активности;
- `TrainingRecentCustomPeriods` хранит последние пять пользовательских периодов;
- `WeeklyEffortChartScale` хранит текущий уровень масштаба и point spacing.

SwiftUI:

- `SportDashboardView` загружает активности и держит общий фильтр;
- `TrainingListView` получает активности, период и выбранные типы через параметры;
- `WeeklyEffortView` получает активности, период, выбранные типы и scale binding через параметры;
- period/type sheets меняют общий state в dashboard.

## Тестирование

Тесты должны покрыть:

- общий фильтр типов объединяет `cycling` и `indoor_cycling` в `Велик`;
- множественный выбор типов включает и исключает активности;
- список тренировок фильтруется одновременно по периоду и типам;
- график фильтруется одновременно по периоду и типам;
- пользовательские периоды сохраняются максимум в количестве пяти и повторный выбор поднимается наверх;
- формат даты оси X `dd/MM/yyyy`;
- подписи X имеют шаг, рассчитанный так, чтобы на экран помещалось максимум четыре подписи;
- `WeeklyEffortChartScale` умеет уменьшать, увеличивать и сбрасывать масштаб.

# Weekly Effort UI Design

## Цель

Сделать первый экран iPhone-приложения SportApp с одной аналитической кривой по Garmin-экспорту. Экран ориентирован на iPhone 15 Pro Max, iPhone Air и iPhone 14 Pro Max.

## Источник данных

Используется Garmin-экспорт из текущего проекта:

```text
from_garmin_official_export/DI_CONNECT/DI-Connect-Fitness/iadomyshev@gmail.com_0_summarizedActivities.json
```

Для работы приложения файл должен быть доступен приложению как bundled resource или как скопированный JSON внутри проекта. Исходную папку `from_garmin_official_export` не изменяем.

## Расчет недельных точек

Гранулярность графика - одна полная неделя строго с понедельника по воскресенье.

Данные в начале и конце диапазона отбрасываются, если они не образуют полную неделю. По текущему файлу это дает 50 полных недель: `2025-06-16` - `2026-05-31`.

Формула:

1. Для каждой тренировки считаем:

```text
trainingValue = calories / durationInMinutes
```

2. Для каждого дня считаем арифметическое среднее значений тренировок этого дня:

```text
dayValue = average(trainingValue внутри дня)
```

3. Для недели складываем значения дней:

```text
weekValue = sum(dayValue с понедельника по воскресенье)
```

Активность не участвует в расчете, если у нее нет `calories`, нет `duration`, или duration равен 0. День без валидных тренировок дает 0.

Фильтр тренировок должен быть выделен отдельной точкой расширения. Начальное правило: исключить `INVALID`, `MEDITATION`, `Rest`, `Track Me`, `Meditation`; остальные валидные активности считать тренировками.

## UI

Экран:

```text
Илья
ты тренируешься уже N недель
[горизонтально прокручиваемая кривая]
```

Текст `N` равен количеству всех доступных полных недель. Пользователь не может уменьшить диапазон статистики.

График:

- горизонтальный ScrollView;
- одна синяя кривая;
- точки на кривой тонкие;
- линия тонкая;
- без подписи `scroll`;
- оптимизация виртуализацией не нужна на этом этапе;
- возможность автоподгрузки при скролле только обозначается в архитектуре, не реализуется.

Tooltip на точке:

```text
Неделя
MM/DD/YYYY
MM/DD/YYYY
```

Все строки tooltip выровнены по центру. Первая дата всегда понедельник, вторая дата всегда воскресенье. Формат дат американский: `mm/dd/yyyy`.

На iPhone вместо hover используется tap/drag gesture по точке графика. Если среда поддерживает pointer hover, можно использовать тот же selected-point state.

## Архитектура

Основные части:

- `GarminActivity` - Decodable-модель нужных полей активности.
- `GarminActivitiesLoader` - чтение JSON из bundle.
- `TrainingActivityFilter` - правило, что считается тренировкой.
- `WeeklyEffortCalculator` - календарная логика, отбрасывание неполных недель и расчет `weekValue`.
- `WeeklyEffortPoint` - модель точки графика: начало недели, конец недели, значение.
- `WeeklyEffortView` - SwiftUI-экран.
- `WeeklyEffortChartView` - отрисовка линии, точек, tooltip и горизонтального скролла.

Календарь должен использовать `Calendar(identifier: .gregorian)` с `firstWeekday = 2`, чтобы неделя начиналась в понедельник независимо от локали устройства.

## Ошибки и пустые состояния

Если JSON не найден, не читается или не содержит валидных полных недель, экран показывает имя `Илья`, строку `ты тренируешься уже 0 недель` и пустую область графика без падения приложения.

## Тестирование

Юнит-тесты на Swift Testing:

- отбрасывание неполных недель в начале и конце;
- начало недели всегда понедельник, конец воскресенье;
- формат tooltip `MM/DD/YYYY`;
- расчет одной тренировки;
- расчет нескольких тренировок в один день через среднее арифметическое;
- расчет недельного значения как суммы дневных значений;
- пропуск активностей без калорий или длительности;
- фильтр исключает `INVALID`, `MEDITATION`, `Rest`, `Track Me`, `Meditation`.

UI-проверка:

- экран открывается на iPhone 15 Pro Max / iPhone Air / iPhone 14 Pro Max simulator;
- отображаются `Илья`, `ты тренируешься уже N недель`, график и tooltip по выбору точки.

# Дизайн: вкладка тренировок и оси графика

## Цель

Переименовать верхние вкладки приложения: `Сегодня` становится `Тренировки`, `История` становится `График`.

Во вкладке `Тренировки` вместо текущей сводки нужно показать список активностей из того же bundled Garmin JSON, который сейчас используется приложением для графика. Список идет от самых новых тренировок сверху к более старым ниже. Вкладка `График` сохраняет текущий исторический график усилия, но получает до шестнадцати подписей по оси X и четыре подписи по оси Y.

## Допущения

- Период по умолчанию для списка тренировок: `1 неделя` от последней доступной активности в bundled Garmin JSON, а не от календарной даты устройства. Это нужно, чтобы исторический экспорт не давал пустой экран, если в нем нет сегодняшних тренировок.
- `Rest` показывается в списке как `Отдых`.
- Список тренировок читает исходные bundled активности. График сохраняет существующую фильтрацию усилия и не начинает включать `Rest` в расчет.
- Формат дат на оси X: `dd/MM/yy`, где день и месяц всегда дополняются ведущим нулем, а год состоит из двух цифр. Примеры: `07/06/26` = 7 июня 2026, `12/04/24` = 12 апреля 2024.
- Дистанция берется из поля Garmin `distance`, если оно есть. В Garmin export это поле хранится в сантиметрах; перед отображением значение переводится в метры и километры. Отсутствующая дистанция отображается как `-`.
- Единица дистанции в Garmin summary — сантиметры. Перед форматированием значение делится на 100; затем используется `км`, если получилось не меньше 1000 метров, иначе `м`.

## Вкладка `Тренировки`

Верхняя панель вкладок:

- слева `Тренировки`;
- справа `График`;
- выбранная вкладка выделяется так же, как сейчас.

Над списком карточек:

- справа или в строке заголовка есть ненавязчивая icon-button настройки периода;
- период по умолчанию `1 неделя`;
- доступные пресеты: `1 неделя`, `2 недели`, `Месяц`, `3 месяца`, `6 месяцев`, `Год`;
- доступен кастомный период через date range;
- выбранный период фильтрует список.

Карточка активности:

- заголовок: человекочитаемый тип тренировки;
- поля по умолчанию: дальность, длительность тренировки, время старта;
- справа сверху маленькая ненавязчивая шестеренка;
- шестеренка открывает настройки визуала для такого типа тренировки;
- в настройках можно выбрать отображаемые поля для этого типа активности.

Названия активностей:

- `open_water_swimming` -> `Плавание Море`;
- `lap_swimming` -> `Бассейн`;
- `cycling` и `indoor_cycling` -> `Велик`;
- имя или тип, содержащий `rest`, -> `Отдых`;
- `running` -> `Бег`;
- `walking` -> `Ходьба`;
- `strength_training` -> `Силовая`;
- `elliptical` -> `Эллипс`;
- прочие типы показываются через название Garmin `name`, если оно не пустое.

## Вкладка `График`

График должен сохранить:

- еженедельную гранулярность понедельник-воскресенье;
- существующий расчет усилия;
- горизонтальный скролл;
- tooltip недели.

Нужно добавить:

- до шестнадцати равномерно распределенных подписей по X;
- при наличии шестнадцати и более точек показываются первая и последняя недели, прежние две промежуточные опорные позиции и еще по четыре даты между каждой соседней парой этих четырех позиций;
- при наличии менее шестнадцати точек подписывается каждая доступная точка без повторяющихся индексов;
- формат подписей X: `dd/MM/yy`;
- подписи остаются горизонтальными; дополнительное скрытие или поворот подписей не добавляются;
- четыре подписи по Y: нижняя, две промежуточные, верхняя;
- Y строится от 0 до максимального значения графика;
- промежуточные Y значения: примерно 1/3 и 2/3 от верхнего значения.

## Архитектура

Доменную логику списка тренировок лучше вынести из SwiftUI:

- `GarminActivity` расширяется необязательным `distance`;
- `TrainingActivityListBuilder` фильтрует период, сортирует активности и строит card models;
- `TrainingActivityPresentation` отвечает за русские названия, формат длительности, дистанции и времени;
- `TrainingActivityFieldPreferences` хранит выбранные поля по типу активности.

SwiftUI слой:

- `TrainingListView` загружает bundled активности, держит период и настройки полей;
- `TrainingActivityCardView` рисует карточку;
- `TrainingPeriodPickerView` открывается по иконке периода;
- `TrainingTypeFieldSettingsView` открывается по шестеренке карточки.

График:

- `WeeklyEffortPoint` получает отдельный formatter для оси X;
- `WeeklyEffortChartView` рисует подписи X/Y внутри текущей горизонтально-скроллируемой области.

## Тестирование

Тесты должны покрыть:

- декодирование `distance`;
- названия `Плавание Море`, `Бассейн`, `Отдых`, `Велик`;
- сортировку списка от новых к старым;
- фильтр периода `1 неделя` от последней активности;
- кастомный date range;
- формат даты оси X `dd/MM/yy`;
- шестнадцать равномерно распределенных значений оси X для длинной истории;
- отсутствие повторяющихся индексов оси X для истории короче шестнадцати недель;
- четыре значения оси Y.
